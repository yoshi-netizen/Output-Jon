# README
## 1. サービス概要

### このサービスはどんなサービスか

自分の考えや感情をうまく言葉にできない人のための、思考整理と言語化をトレーニングするアプリです。

短時間の思考整理ワークを継続することで、頭の中のモヤモヤを整理し、自分の考えを言葉にしやすくなります。

結果として、報告・相談・意見発信などのコミュニケーション力向上を支援します。

### 誰の、どんな課題を、どう解決するか

社会人として働く中で、

- 何を相談すればいいか分からない
- 考えが整理できず、意見をうまく伝えられない
- 言語化力を鍛えたいが、何をすれば良いか分からない

と感じている人の課題を解決するサービスです。

短時間で思考を書き出すワークを継続することで、

**自分の考えを整理し、自然に報連相や意見発信ができる状態を目指します。**

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

このサービスのアイデアは、**自分自身の「報連相ができなかった経験」** から生まれました。

高校生の頃、部活動の先輩が毎日新聞記事を選んで感想を書き、先生がコメントを返す取り組みが行われていました。

当時は「大変そうだな」と思う程度で、特に深く考えることはありませんでした。

しかし社会人になってから、この経験を思い出す出来事がありました。

工場勤務していたある時、私は組み作業のチームから単独作業のチームに異動しました。

異動によって今までと仕事の進め方が変わり、**上司から進捗を確認されることが増えました。**

私は「指示通りに作業しているだけなのに」と感じていましたが、

上司からすると

- 進捗を知りたい
- 想定より作業時間が長い
- 何を考えて行動しているのか分からない

という状態だったようです。

その後、報連相を意識するようになりましたが、

- 何を相談すればいいのか分からない
- どう伝えればいいのか分からない

という状況が続き、悶々とした日々を過ごしました。

そこで書籍や記事を調べ、

PREP法やDESC法などの「伝え方」のフレームワークを学びました。

しかし実際には、

**「そもそも自分の考えが整理できていない」**

という問題に気付きました。

そのとき参考になったのが、書籍の 「0秒思考」 でした。

その本では、1分間で思考を書き出すメソッドが紹介されています。

この方法が自分に合っていた理由は主に3つあります。

- 短時間で実践できる
- 方法がシンプルで続けやすい
- 数回で思考整理の効果を実感できた

それから毎朝このワークを続けるようになりました。

すると少しずつ

- 考えが整理できるようになる
- 報連相がしやすくなる
- 上司とのコミュニケーションが改善する

という変化が起きました。

このとき、ふと高校時代の先輩の取り組みを思い出しました。

「思ったことを書き出す」というシンプルな習慣が、思考の言語化力を鍛えるものだったのだと、そのとき初めて実感しました。

結果として、仕事の進めやすさや職場での関係性も大きく改善しました。

この経験から、

**「思考を言語化するトレーニングを、もっと気軽に続けられる仕組みがあれば、多くの人の役に立つのではないか」**

と考え、このサービスを企画しました。

- 自分の考えをあまり言葉にしないタイプ
- 周囲から「何を考えているか分からない」と言われることがあった

**「仕事をきちんとやりたいのに、コミュニケーションが原因でうまくいかない」**

は仕事をする上で重要なスキルだと感じるようになりました。

### **具体的な利用イメージ**

#### **【例】考えたいテーマ(悩み)の場合**

**1. 考えたいテーマを入力する(ユーザーが自由入力)**

- 学習の進め方

**2. 思考を書き出す(ユーザーが自由入力)**

- 現状、スクールで学習中
- カリキュラムが難しくなってきた
- 何を覚えて、何は覚えなくて良いか分からない
- 分からないことを全て調べた方が良いか
- 時間が足りない
- どこまでAIに頼って良いのか
- 内容を完全に理解しながら進めると到底終わらない
- 講師に相談した方が良いと思う

**3. 思考の核を選択する(ユーザーが選択)**

- 講師に相談した方が良いと思う

**4. 感想や考えを整理する(AI生成)**

現在スクールで学習を進めているが、カリキュラムの難易度が上がり、学習の優先順位付けに苦慮している。「すべてを完全に理解しようとすると時間が足りない」という現実と、「どこまで深掘りすべきか」という境界線が分からず、焦りを感じている状態。

特に、AIをどの程度活用すべきか、また「何を覚え、何を調べればよいか」という判断基準が自分一人では持てていないことが大きな課題。

そのため、一度講師の方に現状を相談し、実務で求められる「必須知識」と「都度調べればよい知識」の切り分けについてアドバイスを頂きたいと考えている。相談を通じて、限られた時間の中で着実にスキルを身につけるための、自分に合った「学習のペースメーカー」を再構築することを目指す。

  
#### **【例】考えたい記事リンク(外部記事のインプット)の場合**

**1. 考えたい記事リンクを入力する(ユーザーがリンク先からコピーして入力)**

- `https://search.app/example`(例として仕様駆動開発についての記事とします)

**2. 思考を書き出す(ユーザーが自由入力)**

- 仕様駆動開発の体験と個人的なベストプラクティスを提供してくれる記事
- 仕様駆動開発をあまり分かってなかったがこの記事を機会に調べて少し掴めた
- めちゃくちゃ簡単に言うと設計を作り込んでAIに作ってもらうものと理解
- ドキュメントが要らないとの主張は驚きであり目から鱗だった
- web開発初学者にはハードルが少し高いと感じた
- コードを見て仕様を理解出来ないため
- AIを使いこなしたい
- 前提でコードリーディングスキルが必要
- 別の発信者がAI利用の学習に注意を促していた
- 学習には適度に摩擦が必要
- オススメ本としてプログラマー脳
- 本読みたい
- 結局どうすればいい？

**3. 思考の核を選択する(ユーザーが自由入力)**

- 結局どうすればいい？

**4. 感想や考えを整理する(AI生成)**

「仕様駆動開発」という概念に触れ、AIに設計を委ねるための前提条件が明確になりました。特に「ドキュメント不要」という考え方は衝撃的でしたが、それを成立させるには高度なコードリーディングスキルが不可欠であると痛感しています。

Web開発初学者にとって、AIは強力な武器になる一方で、自力でコードを理解できないまま進むことへのリスクも感じました。別の発信者が指摘していた「学習には適度な摩擦が必要」という言葉通り、安易にAIに頼り切るのではなく、基礎体力を養うフェーズを大切にしたいと考えています。

今後の具体的なアクションとして、まずは推奨されていた書籍『プログラマー脳』を読み、構造的にコードを捉える力を養います。AIを真に使いこなすために、まずは「コードを読み解く脳」を鍛えることから始めていく決意です。

## 9. 今後の展開・発展の方向性

このアプリは、**「投稿して終わり」にならないために**、単なるメモツールから「思考のパーソナルトレーナー」へと進化させます。

### 観点

- **ユーザーはなぜ使い続ける？**
    
    「書けば頭が整理される」という成功体験に加え、カレンダーによる「積み上げの可視化」が自己肯定感を高める報酬（フィードバック）になるためです。
    
- **続けることで何が変わる？**
    
    瞬発的な言語化力が鍛えられ、会議や報告の場で「えーっと」と詰まる時間が減り、自分の意見をロジカルに伝える自信がつきます。
    
- **データが溜まると何ができる？**
    
    「自分が何に悩みやすいか」「どんなテーマだと筆が進むか」といった思考のバイアス（傾向）をAIが分析し、自己理解を深めるレポートを提供できます。
    
- **他の人と関わる余地はある？**
    
    「同じニュース記事」に対して、他のユーザーがどのような「核」を選び、どう言語化したかを見ることで、多角的な視点を取り入れる「学び合い」が生まれます。
    

### 今考えている発展案

- **機能追加の方向性：**
    - **用途に合わせた柔軟な出力に対応:** 整理した内容を、そのままSlackやメールの「相談報告用テンプレート」として書き出す機能。
    - **言語化レベル測定機能:** 言語化レベルの測定する機能。
- **UI・体験の改善：**
    - **ゲーミフィケーション:** 継続日数に応じたランクアップ機能。
    - **集中を削がないミニマリズム:** 入力中は残り時間以外のノイズを極限まで削ぎ落とし、ゾーンに入りやすくするUI。
- **機能以外の工夫：**
    - **心理的安全性の確保**:「正しい日本語でなくてOK」「主語がなくてもOK」「箇条書きで単語だけでもOK」といった、ハードルを下げるメッセージを常に表示させる。
    - **「思考整理ワーク」の公式ガイド**:「うまく書こうとしない」「語尾を気にしない」といった、このアプリを使いこなすためのマインドセット（心得）をチュートリアルやコラムで伝える。

### 10-2 技術選定理由

### なぜこの技術を使うのか

本アプリは

- ユーザー登録
- 投稿保存
- フォーム入力
- AI生成
- 投稿一覧表示

といった **CRUD中心のWebアプリケーション**であるため、開発スピードと実装の安定性を考慮してRuby on Railsを採用しました。

フロントエンドは複雑なSPA構成を避け、Rails標準のHotwireを使用することで、JavaScriptの実装量を抑えながらインタラクティブなUIを実現します。

スタイリングにはTailwind CSSを使用し、シンプルで保守しやすいUI構築を行います。

AIによる言語化サポート機能はGemini APIを利用して実装予定です。

---

### 今回チャレンジしたい点

- Gemini APIを利用したAI文章生成機能の実装
- OGP取得による記事リンクの情報表示
- 思考整理ワークをステップ形式で進めるUI設計
- 継続利用を意識したログ管理機能の設計

### **Gemini API実装イメージ**

**【公式ドキュメントより】**

- APIでテキスト生成可能
- 実装手順

  1.APIキー作成
  
  2.Google GenAI SDKをインストール
  →Rubyはサポートなし
  
  3.リクエスト送信
  
- REST APIはどの言語でも可能
  
**【解説記事より】**

- RubyではSDKの代わりにNet::HTTPでAPIリクエスト送れる(詳細解説あり)
  
**【本アプリでのリクエストタイミング】**

1. 考えたいテーマや記事リンクを入力する
  →フォーム入力
2. 思考を書き出す
  →フォーム入力
3. 思考の核を選択する
  →MVPではフォーム入力予定
4. 感想や考えを整理する
  →**このタイミングでAI生成ボタンをトリガーにフォーム入力情報を「まとめて」リクエスト送信**
